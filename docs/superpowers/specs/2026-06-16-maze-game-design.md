# 迷宫游戏设计规格

## 概述

为游戏控制台新增迷宫小游戏（`maze`）。每次打开用递归回溯 DFS 生成全新迷宫，可通过种子复现。玩法为“收集 + 逃脱”：从起点走到出口，沿途收集散落在死胡同里的宝石。

## 迷宫生成

**算法**：递归回溯（Recursive Backtracking / DFS），显式栈实现。

- 网格：30 列 × 35 行（1050 格），每格记录四面墙状态（N/S/W/E 位标记）
- 初始所有格子四面有墙
- 从 (0,0) 出发，随机选未访问邻居，打通中间墙壁，压栈前进
- 走到死路时弹栈回溯
- 重复直到全部格子都被访问
- 生成的长走廊蜿蜒曲折，岔路少但深，走起来有探索感

**随机数**：沿用项目现有的内联 LCG 模式：
```c
g_rand_state = g_rand_state * 1664525u + 1013904223u;
```
每次开局从 `Bsp_Get_Tick_Ms()` 取种子，确保每次不同。相同种子 → 相同迷宫。

**内存预算**：约 4 KB
- 墙壁网格：1050 字节
- 访问标记（仅生成时用）：1050 字节（可用墙壁字节的高位复用，省掉）
- DFS 栈：最坏 1050 × 2 = 2100 字节

## 玩法

### 状态机
- `maze_state_ready` — 迷宫已绘制，等待开始
- `maze_state_playing` — 玩家移动中
- `maze_state_over` — 到达出口，显示得分

### 流程
1. **Ready**：显示完整迷宫 + "PUSH UP TO START" 提示
2. **Playing**：摇杆每推一次移动一格，墙壁阻挡不可通行。碰触宝石即收集。顶栏实时更新分数。
3. **Over**：到达出口 → 加通关奖励 → 显示总分 → "GAME OVER" → 排行榜 → 可选重玩或返回菜单

### 宝石放置
- 迷宫生成后，BFS 找最短路径
- 宝石放在死胡同（只有一面开口的格子）和岔路深处，**不放在**最短路径上
- 约 60-70% 的死胡同有宝石
- 鼓励玩家离开主路探索

### 操作
| 输入 | 行为 |
|------|------|
| 摇杆方向 | 玩家移动一格（有墙则不动） |
| 长按按钮 1 秒 | 返回菜单 |
| 任意方向（游戏结束） | 重新开始 |

## 屏幕布局

```
┌──────────────────────────┐ y=0
│ MAZE  SEED:12345  💎5    │ 顶栏 12px
├──────────────────────────┤ y=12
│                          │
│    30×35 迷宫区域        │ 280px（35 格 × 8px）
│    （每格 8×8 像素）     │
│                          │
├──────────────────────────┤ y=292
│        （6px 间隔）       │
├──────────────────────────┤ y=298
│    HOLD TO BACK          │ 底栏 22px
└──────────────────────────┘ y=320
```

- 迷宫区域：240×280 px，起点 y=12
- 底部 6px 黑色间隔（y=292-298），然后底栏

## 视觉效果

### 颜色（RGB565）
| 元素 | 颜色 | 值 |
|------|------|-----|
| 墙壁 | 深灰蓝 | `0x4208` |
| 地板/通道 | 黑色 | `0x0000` |
| 玩家 | 青色 | `0x07ff` |
| 宝石 | 黄色 | `0xffe0` |
| 出口 | 红色 | `0xf800` |
| 顶栏/底栏背景 | 黑色 | `0x0000` |
| 文字 | 白色/灰色 | `0xffff` / `0x8410` |

### 绘制策略
- **一次性绘制**：初始化时画完整迷宫。先用墙壁色填充整个迷宫区域，再给每格内部填地板色（6×6 留 1px 墙），最后打通相邻连通格之间的墙壁。
- **逐帧增量**：擦除玩家旧位置（填黑），画玩家新位置（6×6 青色方块）。收集宝石时擦除（填黑）。更新顶栏分数。
- 全部使用 `Game_Graphics_Fill_Rect()` 绘制
- 文字用 `Game_Graphics_Draw_Text()` / `Game_Graphics_Draw_U32()`

## 计分

- 每颗宝石：**+100**
- 到达出口：**+500 奖励**
- 总分 = 宝石分 + 通关奖励

## 种子系统

- 顶栏显示 `SEED:12345`
- 每次从 `Bsp_Get_Tick_Ms()` 生成种子
- 玩家可记下种子号以便复现喜欢的迷宫
- 调试时可在 `restart_game()` 中硬编码种子做可复现测试
