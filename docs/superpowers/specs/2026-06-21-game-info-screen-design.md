# 菜单游戏介绍界面设计

## 目标

菜单选中任意注册项时按 X 打开简短介绍页。页面包含 DESCRIPTION、GOAL、CONTROLS 三段，B 返回菜单。顶部和底部沿用统一状态栏风格。

## 数据与资源约束

`Game_descriptor` 增加一个 `const char* info_text`。每个注册项在 `game_registry.c` 中直接提供只读字符串；字符串用显式 `\n` 手动换行，连续 `\n` 产生空行。正文每行最多 34 个字符，适配 240 px 屏幕左右 12 px 边距与 6 px 字符宽度。

除字符串外不建立说明对象、页面数组或堆分配。渲染时仅使用 35 字节栈上行缓冲；新增状态和逻辑保持常量级 RAM，占用主要来自 Flash 中的介绍文字。

## 界面与交互

- 菜单底栏显示 `A OK  X INFO  B BACK`，菜单内容区保留摇杆提示并补充 `X: INFO`。
- 介绍页顶部显示当前条目名；正文区域为带边框的深色卡片。
- DESCRIPTION、GOAL、CONTROLS 标题使用青色，正文使用白/灰色，段落空行自然留白。
- 底栏显示 `B BACK` 与 FPS。
- B 返回菜单；介绍页也参与现有 30 秒屏保，唤醒后重绘介绍页。
- A/Y/方向键在介绍页不执行动作，避免误启动游戏。

## 架构

新增纯函数模块 `game_info_text.c/.h` 负责按 `\n` 拆行，`game_info_screen.c/.h` 负责绘制，两者均不持有状态。`game_console.c` 增加 `console_state_game_info`，负责 X 入口、B 返回、FPS 更新、屏保恢复。注册表是介绍内容的唯一来源。

## 验证

- 单元测试覆盖普通换行、连续换行和末尾文本。
- 注册表测试确保每个条目都有三段介绍，且每个显示行不超过 34 字符。
- 状态路径检查确保 X 从菜单进入，B 返回，屏保恢复能重绘。
- VM 全量构建通过。
